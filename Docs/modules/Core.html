ArtifactInterface.lua

Provides utilities and manages interaction with Blizzard's ArtifactUI (as well as Artifact Knowledge/Research Notes, which are accessed via GarrisonUI)
GetArtifactKnowledgeLevel ()
Returns the the current AK level (same as used in Blizzard's Forge tooltip)

Returns:

    Current Artifact Knowledge Level
GetNumAvailableResearchNotes ()
Returns the number of Artifact Research Notes that are ready for pickup

Returns:

  1. Amount of Research Notes that are available for pickup
  2. Maximum number of shipments that have been queued
GetNumRanksPurchasableWithAP (rank, power, tier)
Returns the number of traits that can be purchased

Parameters:

  • rank The current artifact level (number of purchased traits)
  • power Number of unspent artifact power applied to the weapon
  • tier Artifact tier (defaults to 1; new tiers unlock additional traits)

Returns:

    Number of traits that can be purchased
GetProgressTowardsNextRank (rank, power, tier)
Returns progress towards the next trait (after considering all available "level ups")

Parameters:

  • rank The current artifact level (number of purchased traits)
  • power Number of unspent artifact power applied to the weapon
  • tier Artifact tier (defaults to 1; new tiers unlock additional traits)

Returns:

    Percentage towards the next available trait (after buying as many as possible)
GetResearchNotesShipmentInfo ()
Returns information about the currently queued Artifact Research work order status

Returns:

  1. Localized name of the work order item (this is "Artifact Research Notes" in English)
  2. Localized string representing the time left until the next work order is available for pickup
  3. Duration (in seconds) until the next work order is available for pickup
  4. Time (in seconds) that has elapsed since the current work order has been started
  5. Number of work orders that are available for pickup
  6. Total number of work orders that are currently in progress
  7. Name of the item (That, too, is "Artifact Research Notes" in English)
GetTimeUntilNextResearchNoteIsReady ()
Returns the time (integer timestamp, String) until the next Artifact Research Notes are ready to be picked up

Returns:

  1. Localized string representing the time left until the next work order is available for pickup
  2. Duration (in seconds) until the next work order is available for pickup

Cache.lua

Provides an interface for the addon's progress cache (stored via SavedVars).
GetDefaults ()
Returns the base structure for an "empty" cache entry. It contains values for a spec that hasn't been scanned yet, where all values are nil except "isIgnored" (which is FALSE) This is so that specs that haven't been cached yet can be detected, but won't cause errors when their cache entries are being accessed

Returns:

    A newly-created table with predefined keys and the setting for "ignore spec" disabled

Usage:

    GetDefaults() -> { ["numTraitsPurchased"] = nil, ["thisLevelUnspentAP"] = nil, ["artifactTier"] = nil, ["isIgnored"] = false }
GetEntry (fqcn, specID)
Returns the entire cache entry for a given character and spec

Parameters:

  • fqcn Fully-qualified character name, to be used as the key
  • specID Specialization ID, to be used as the secondary key

Usage:

    GetEntry("Duckwhale - Outland", 1) ->  { ["numTraitsPurchased"] = 15, ["thisLevelUnspentAP"] = 235000, ["artifactTier"] = 1, ["isIgnored"] = false }
GetNumIgnoredSpecs ([fqcn])
Returns the number of ignored specs for a given character (defaults to currently used character if none is given)

Parameters:

  • fqcn Fully qualified character name, to be used as the primary key (optional)

Returns:

    Number of ignored specs; 0 if none are cached
GetReference ()
Returns a reference to the underlying SavedVars (cache) object
GetValue (fqcn, specID, key)
Returns the cached value of the respective character and spec for a given key

Parameters:

  • fqcn Fully-qualified character name, to be used as the primary key
  • specID Specialization ID, to be used as the secondary key
  • key The key used to look up values inside of the cache entry

Usage:

    GetValue("Duckwhale - Outland", 3, "numTraitsPurchased") -> 15
NewEntry (fqcn, specID[, defaults])
Add a new cache entry for the respective character/spec, and optionally sets it to predefined values (uses empty "default" entry if none was given)

Parameters:

  • fqcn Fully-qualified character name, to be used as the primary key
  • specID Specialization ID, to be used as the secondary key
  • defaults A table containing default entries that should be used (optional)

Usage:

    NewEntry("Duckwhale - Outland", 2) ->  { ["numTraitsPurchased"] = nil, ["thisLevelUnspentAP"] = nil, ["artifactTier"] = nil, ["isIgnored"] = false }
UpdateEntry (fqcn, specID, updateValues)
Updates an existing entry for the respective character and spec with the given values

Parameters:

  • fqcn Fully qualified character name, to be used as the primary key
  • specID Specialization ID, to be used as the secondary key
  • updateValues A table representing the new entry, to replace any existing entry with

Returns:

    true if the update was successful; nil otherwise

Usage:

    UpdateEntry("Duckwhale - Outland", 3, { ["numTraitsPurchased"] = 15, ["thisLevelUnspentAP"] = 235000, ["artifactTier"] = 1, ["isIgnored"] = false }  -> true

DB.lua

Provides an interface for the addon's static database (i.e., all files stored in the \DB\ folder)
GetArtifactItemID (classID, specID)
Returns the item ID of the artifact weapon for the given class and spec

Parameters:

  • classID The class identifier (as returned by UnitClass("player")). Usually, this is a number between 1 and 12
  • specID The specialization identifier (as returned by GetSpecialization()). Usually, this is a number between 1 and 4

Returns:

    Item ID of the corresponding artifact weapon; NIL if no DB entry exists
GetArtifactWeapons ()
Returns a reference to the artifact weapons DB object. For internal use only
GetDB ()
Returns a reference to the currently used SavedVars (DB) object. Temporary crutch for ongoing refactoring/migration of modules :)
GetDefaults ()
Returns the default settings for the initial startup or manual/automatic resets during verification. Temporary crutch for ongoing refactoring/migration of modules :)
GetItemEffectID (itemID)
Returns the item spell effect ID for a given itemID

Parameters:

  • itemID The identifier for a specific artifact power item

Returns:

    ID of the spell effect corresponding to the given artifact power item's "Empowering" spell (which is how AP is applied to the currently equipped artifact ingame); NIL if the itemID was invalid or no DB entry exists
GetItemEffects ()
Returns a reference to the item effects DB (spell effects). For internal use only
GetReference ()
Returns a reference to the top-level DB object. Temporary crutch for ongoing refactoring/migration of modules :)
RestoreDefaults ()
Reset settings to their default values. Overwrites the SavedVars part responsible for storing the user's settings
UnignoreAllSpecs ([fqcn])
Removes all specs from the ignored specs list for a given character (defaults to currently used character if none is given)

Parameters:

  • fqcn Fully-qualified character name that will have their specs "IsIgnored" setting reset (optional)

Scanner.lua

Handles scanning of spell descriptions for artifact power tokens
ParseSpellDesc (spellDescription[, localeString])
Scans spell description and extracts AP amount based on locale (some use different formats to display numbers)

Parameters:

  • spellDescription The localized spell description text as returned by the WOW API
  • localeString The locale in the typical format (enUS, deDE, frFR, ...). Defaults to currently active locale if omitted (optional)

Returns:

    The number of artifact power that this spell will apply to the current artifact
RegexEscapeChar (c)
Looks up string-characters to find their regexp pattern string (purely for ease of use and readability)

Parameters:

  • c The character that the pattern should represent

Returns:

    The Lua-expression pattern that can be used to match this character
